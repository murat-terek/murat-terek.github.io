1. RPO и RTO - Что это такое и в чём разница - За 5 Минут - Вопросы с Интервью SysAdmin DevOps

 

RPO   - Recovery Point Objective. RPO = DATA LOSS = Время -> сколько информации в временном выражении можно потерять при Disaster (авария, все сгорело). Например, последние 1 час или 15 минут. Время с момента Backup до Disaster.

RTO   - Recovery Time Objective. RTO = DOWNTIME = Время -> сколько времени нужно, чтобы восстановить работоспособность системы при Disaster (авария, все сгорело).     Например, 30 минут,  1 час или 2 часа. Время с Disaster до полного восстановления.

RPO 1 час   - Backup каждый час

Создание Replica (копии) поможет уменьшить потерю данных RPO

RTO 6 часов   - ручное создание серверов, ручные настройки

RTO 1 час   - автоматическое создание серверов, Auto Scaling

Использование Multi Region в Cloud поможет уменьшить RTO

Чем меньше RPO/RTO, тем дороже эта инфраструктура

2. PaaS SaaS IaaS - Что это такое и в чём разница - За 5 Минут - Вопросы с Интервью DevOps

 

IaaS   – Infrastructure as a Service. Виртуальные сервера, базы данных, компоненты сети, хранилища. Вы занимаетесь настройкой всего сами, включая код. Например: AWS EC2 (виртуальные сервера), AWS RDS (базы данных в облаке), AWS VPC (виртуальные сети), Google Compute Engine (виртуальные сервера).

PaaS   – Platform as a Service. Вы занимаетесь только кодом, настройка инфраструктуры автоматическая. Например, Google App Engine, AWS App Runner, AWS Elastic Beanstalk, AWS Lambda, Google Cloud Functions.

SaaS   – Software as a Service. Вы занимаетесь только использованием, ничего не нужно настраивать. Например: Dropbox, GMAIL, Google Docs, Google Sheets.

4. Основы Шифрования за 5 минут - Asymmetric Cryptography

 

Public Key (открытый ключ)   - с этим ключом происходит шифрование, может иметь доступ любой.

Private Key (закрытый ключ)   - с этим ключом происходит дешифрование, нужно надежно хранить.

ssh-keygen -t rsa -b 4096 -m PEM -f cryptoKey   - создать публичный и приватный ключи (тип - rsa,      длина — 4096, формат — PEM (текстовое представление)) и записать в файлы (cryptoKey – приватный,  cryptoKey.pub – публичный)

ssh-keygen -f cryptoKey.pub -e -m PKCS8 > cryptoKey.public   - сконвертировать публичный ключ созданный ssh-keygen (cryptoKey.pub) в формат понятный OpenSSL – создается новый файл  cryptoKey.public

rm cryptoKey.pub   - удалить файл cryptoKey.pub (публичный ключ, созданный ssh-keygen, до этого сконвертировали в  PKCS8 формат понятный  OpenSSL – создался новый файл cryptoKey.public и cryptoKey.pub уже не нужен)

mv cryptoKey cryptoKey.private   - переименовать файл cryptoKey (приватный ключ созданный ssh-keygen) в файл  cryptoKey.private (для наглядности)

openssl rsautl -encrypt -pubin -inkey cryptoKey.public -in message.txt -out EncryptedMessage.txt   - зашифровать файл message.txt в файл EncryptedMessage.txt, используя публичный ключ  cryptoKey.public

openssl rsautl -decrypt -inkey cryptoKey.private -in EncryptedMessage.txt -out DecryptedMessage.txt   - расшифровать файл  EncryptedMessage.txt в файл  DecryptedMessage.txt, используя приватный ключ  cryptoKey.private

DHCP (расшифровка)   - Dynamic Host Configuration Protocol (протокол динамической настройки узла)

DHCP сервер   - присваивает клиентам IP адреса (и другие параметры), диапазон которых администратор может заранее задать

APIPA (расшифровка)   - Automatic Private IP Addressing (автоматическая частная IP-адресация)

APIPA (как работает)   - с помощью APIPA клиенты DHCP могут получать IP-адреса, даже если серверы DHCP не работают. При сбое процесса DHCP Windows автоматически назначает IP-адрес из частного диапазона (169.254.0.0/16)

 

Частные диапазоны адресов IPv4   -

·         10.0.0.0 — 10.255.255.255 (маска подсети для бесклассовой (CIDR) адресации: 255.0.0.0 или /8)

·         100.64.0.0 — 100.127.255.255 (маска подсети 255.192.0.0 или /10) - Данная подсеть рекомендована согласно RFC 6598 для использования в качестве адресов для CGN (Carrier-Grade NAT).

·         127.0.0.0 — 127.255.255.255 (маска подсети: 255.0.0.0 или /8) – Данный диапазон  зарезервирован для петлевых интерфейсов (не используется для обмена между узлами сети)

·         172.16.0.0 — 172.31.255.255 (маска подсети: 255.240.0.0 или /12)

·         192.168.0.0 — 192.168.255.255 (маска подсети: 255.255.0.0 или /16)

в конце файла конфигурации HAProxy прописать, чтобы при обращении на порт 80 с любого IP адреса шло перенаправление на сервера 172.31.111.111 при 90% обращениях и 172.31.222.222 при 10% обращениях и при обращении на порт 8080 с любого IP адреса шло перенаправление на сервер 172.31.333.333   -

/etc/haproxy/haproxy.cfg:

 

#----------------------------------------

frontend my_http_front

   bind *:80

   default_backend my_http_back

 

frontend my_secret_front

   bind *:8080

   default_backend my_secret_back

 

backend my_http_back

   balance roundrobin

   server myweb1 172.31.111.111:80 weight 90

   server myweb2 172.31.222.222:80 weight 10

 

backend my_secret_back

   server mysecretweb 172.31.333.333:80

#----------------------------------------

12. Что такое VPN - Для чего его используют - Установка и Настройка Бесплатного OpenVPN Сервера с нуля

 

VPN (расшифровка)   - Virtual Private Network – виртуальная частная сеть

VPN – это…   - это совокупность технологий, позволяющих обеспечить одно или несколько сетевых соединений (логическую сеть) поверх другой сети (например, Интернет)

 

VPN (графическое изображение)   -

CA (расшифровка)   - Certificate Authority - центр сертификации

Частный ЦС позволяет…   -  выпускать сертификаты для пользователей, серверов или отдельных программ и служб в вашей инфраструктуре

Сколько серверов нужно для работы OpenVPN?   - рекомендуется использовать 2 сервера — один для сервера OpenVPN, а второй в качестве центра сертификации, хотя технически можно и все на одном разместить, но это открывает ваш VPN для ряда уязвимостей

На серверах OpenVPN и CA можно ли обойтись только пользователем root?   - нет, на обоих серверах нужно создать пользователя без прав root с привилегиями sudo

12.1.3. VPN. Центр сертификации. Создание Центра сертификации (Ubuntu 20.04)

 

cp vars.example vars   - скопировать файл vars.example в файл vars (~/easy-rsa/) на сервере CA

 

~/easy-rsa/vars (содержимое файла, который необходимо создать перед созданием корневой пары закрытого и открытого ключей для CA, выделенные значения нужно поменять для отражения информации об организации)   -

set_var EASYRSA_REQ_COUNTRY    "RU"

set_var EASYRSA_REQ_PROVINCE   "KBR"

set_var EASYRSA_REQ_CITY       "Terek"

set_var EASYRSA_REQ_ORG        "MuraSoft"

set_var EASYRSA_REQ_EMAIL      "xxxxxx@mail.ru"

set_var EASYRSA_REQ_OU         "IT"

set_var EASYRSA_ALGO           "ec"

set_var EASYRSA_DIGEST         "sha512"

 

./easyrsa build-ca   - создание корневой пары открытого и закрытого ключей для Центра сертификации в директории ~/easy-rsa/ (нужно будет задать ПАРОЛЬ)

./easyrsa build-ca nopass   - создание корневой пары открытого и закрытого ключей для Центра сертификации в директории ~/easy-rsa/ (НЕ нужно будет задавать ПАРОЛЬ)

~/easy-rsa/pki/ca.crt   - файл публичного сертификата ЦС. Пользователи, серверы и клиенты будут использовать этот сертификат для подтверждения единой сети доверия. Копия этого файла должна иметься у всех пользователей и серверов, использующих ваш ЦС. Все стороны будут использовать публичный сертификат для подтверждения подлинности системы и предотвращения атак через посредника.

~/easy-rsa/pki/private/ca.key   - закрытый ключ, который ЦС будет использовать для подписания сертификатов серверов и клиентов. Если злоумышленник получит доступ к ЦС и файлу ca.key, вам нужно будет уничтожить ЦС. Поэтому файл ca.key должен храниться только на компьютере ЦС, и для дополнительной безопасности компьютер ЦС следует выключать, когда он не используется для подписания запросов сертификатов.

OUTPUT_DIR=~/client-configs/files

curl -   утилита командной строки для Linux или Windows, представляет собой набор библиотек, в которых реализуются базовые возможности работы с URL страницами и передачи файлов

curl -L adv-it.net   -  (curl) выполняет HTTP-запрос get и отображает статическое содержимое страницы. Если есть перенаправление, то выводится конечная страница (НЕ страница перенаправления).

14. SQL Server - Различие Выпусков и Цены на Лицензии (2022)

 

SQL Server – Express Edition (2022)   - самая простая версия SQL Server, полностью бесплатная (FREE), максимальный размер базы данных 10Gb, CPU Cores – 4, Memory – 1410Mb

SQL Server – Web Edition (2022)   - версия SQL Server для хостеров, провайдеров и поставщиков облачных услуг по модели SaaS, может быть использована только для публичных, доступных из интернета вэб приложений или сервисов, не бесплатная (цену определяет только Hosting Provider), максимальный размер одной базы данных 524PB, CPU Cores – 16, Memory – 64Gb

SQL Server – Standard Edition (2022)   - самая распространенная версия SQL Server, для средних размеров приложений, не бесплатная (CPU Core (2 core pack – 3,945$) or Server (989$) + CAL (230$)), максимальный размер одной базы данных 524PB, CPU Cores – 24, Memory – 128Gb

SQL Server – Enterprise Edition (2022)   - самая крутая версия SQL Server, для mission-critical приложений, не бесплатная (CPU Core (2 core pack – 15,123$)), максимальный размер одной базы данных 524PB, CPU Cores – Unlim, Memory – Unlim

SQL Server – Developer Edition (2022)   - все тоже самое как и SQL Server – Enterprise Edition, но полностью бесплатная (FREE), ограничено для использования в NON-PRODUCTION, максимальный размер одной базы данных 524PB, CPU Cores – Unlim, Memory – Unlim

Server + CAL (за что платить - SQL Server (2022))   - Server – цена за одну установку SQL Server, CAL (client access licenses) – цена за каждого пользователя SQL Server

Per Core (за что платить - SQL Server (2022))   - цена за каждый Physical CPU Core или Virtual CPU
