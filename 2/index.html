1. Linux для Не Начинающих - Добавление второго диска

 

/dev/sda    - первый SATA диск

/dev/sdb    - второй SATA диск

/dev/sdc    - третий SATA диск

 

fdisk –l,   lsblk    -  показать какие есть диски

cfdisk   /dev/sdb  - редактировать разделы sdb диска

mkfs.ntfs  –f  /dev/sdb1 - форматировать в NTFS первый раздел второго диска

/etc/fstab   - тут прописываем новые диски чтобы они присоединялись автоматически при загрузки Линукса

 

mount /media/hdd2   - присоединяет диск прописанный в файле fstab с именем hdd2 без перезагрузки Линукса

 

---

umount /dev/nvme0n1p6  - отмонтировать диск (SSD MVMe - nvme0n1p6)

 

---

 

  umount /dev/sda3

  sudo ntfsfix /dev/sda3

  mount /dev/sda3

 

Если при попытке монтирования диска sda3 с файловой системой NTFS вы получаете:

«The disk contains an unclean file system (0, 0).

Metadata kept in Windows cache, refused to mount......»

Диск монтируется, но только для чтения.

 

 

 

2. Linux для Не Начинающих - Изменение имени компа и IP на постоянный

 

hostname    - вывести название компа

sudo hostname MyLinux   - присвоить имя компа «MyLinux»

/etc/hostname   - тут прописано название компа

/etc/hosts  - тут прописываем новое название напротив IP адреса

sudo systemctl restart networking  - перезапустить сеть

ifconfig   - вывести текущий IP адрес

 

sudo  ifconfig  eth0  10.10.10.10  netmask  255.0.0.0    -временно меняем IP адрес

 

До 17.10 версии:

/etc/network/interfaces  - тут прописываем постоянный IP адрес

 

auto lo

iface lo inet loopback

 

auto enp0s3

iface enp0s3 inet static

  address 192.168.1.123

  netmask 255.255.255.0

  gateway 192.168.1.1

  dns-nameservers 8.8.8.8, 77.88.8.8

 

 

sudo ifdown eth0    - отключить сетевуху eth0

sudo ifup   eth0      - включить сетевуху eth0

---

С 17.10 версии:

/etc/netplan/xxx.yaml   - тут прописываем постоянный IP адрес

 

примерное содержимое (с 17.10 версии Ubuntu):

network:

  version: 2

  renderer: NetworkManager

  ethernets:

    enp7s0:

      dhcp4: no

      addresses: [ 192.168.1.123/24 ]

      gateway4: 192.168.1.1

      nameservers:

        addresses: [ 8.8.8.8, 77.88.8.8 ]

 

примерное содержимое (с 22.04 версии Ubuntu, новый синтаксис):

  renderer: NetworkManager

  ethernets:

    enp7s0:

      dhcp4: no

      addresses:

        - 192.168.1.123/24

      routes:

        - to: default

          via: 192.168.1.1

      nameservers:

        addresses:

          - 8.8.8.8

sudo netplan apply   - применяем измененные параметры сети

4. Linux для Не Начинающих - Выполнение по расписанию - crontab

 

crontab - l    - показать расписание

crontab -e   - редактировать расписание

/etc/crontab  - файл расписания на системном уровне

sudo cat /var/spool/cron/crontabs/denis  - уровень пользователя (crontab)

sudo cat /var/log/syslog | grep CRON    - лог crontab

 

* * * * * command(s)

| | | | |

| | | | ----- Day of week (0 - 7) (Sunday=0 or 7)

| | | ------- Month (1 - 12)

| | --------- Day of month (1 - 31)

| ----------- Hour (0 - 23)

------------- Minute (0 — 59)

 

Первые пять полей могут содержать одно или несколько значений, разделенных запятой или диапазон значений, разделенных дефисом.

 

    * - оператор звездочки означает любое значение или всегда. Если в поле «Час» имеется символ звездочки, это означает, что задание будет выполняться каждый час.

    , - оператор запятой позволяет указать список значений для повторения. Например, если у вас есть 1,3,5 в поле Час, задание будет выполняться в 1, 3 и 5 часов утра.

    - - оператор дефиса позволяет указать диапазон значений. Если в поле «День недели» указано значение 1–5, задание будет выполняться каждый рабочий день (с понедельника по пятницу).

    / - оператор косой черты позволяет указать значения, которые будут повторяться в течение определенного интервала между ними. Например, если в поле «Час» указано */4, это означает, что действие будет выполняться каждые четыре часа. Это то же самое, что указание 0,4,8,12,16,20. Вместо звездочки перед оператором косой черты можно также использовать диапазон значений, 1-30/10 означает то же, что и 1,11,21.

5. Linux - Использование SSH ключа вместо Пароля | SSH Key Pair

 

ssh-keygen (описание)  - это утилита командной строки, используемая для генерации и управления ключами SSH

ssh-keygen (запуск без ключей)   - запросит имя и пароль на ключ (пароль можно пропустить), на выходе будет приватный ключ с указанным именем, но без всякого расширения, а публичный будет так же с этим именем, но с расширением .pub)

ssh-keygen (в любом месте, но сейчас сделаем на главном в ~/demo/, указать имя mykey, без пароля)   - создастся приватный ключ ~/demo/mykey и публичный ключ  ~/demo/mykey.pub

scp ~/demo/mykey.pub murat@172.16.0.11:/tmp - скопировать публичный ключ ~/demo/mykey.pub на сервер 172.16.0.11 в папку /tmp, авторизовавшись под именем murat

cat /tmp/mykey.pub >> ~/.ssh/authorized_keys   - добавить ранее скопированный публичный ключ /tmp/mykey.pub в конец файла ~/.ssh/authorized_keys (если его не существует, то он создастся)

chmod 0600 ~/.ssh/authorized_keys   - установить только права RW владельцу файлу authorized_keys

chmod 0600 ~/.ssh/id_rsa   - установить только права RW владельцу файлу id_rsa

ssh -i ~/demo/mykey murat@172.16.0.11   - подключение по ssh к 172.16.0.11 (user – murat), используя приватный ключ ~/demo/mykey

~/.ssh/known_hosts   - файл где хранятся открытые ключи серверов (компьютеров), к которым подключались ранее (если уже один раз подключились, то не будет ни каких вопросов при следующих подключениях)

~/.ssh/id_rsa   - файл с приватным ключем, который система использует по умолчанию (если он есть, то можно подключаться по ssh, не указывая ключ и пароль соответствующего удаленного сервера)

6. Linux - Использование множества SSH ключей для подключения на разные Linux | ssh config

 

Глобальный файл конфигурации SSH в Linux   - /etc/ssh/ssh_config

Пользовательский файл конфигурации SSH в Linux   - ~/.ssh/config

Глобальный файл конфигурации SSH в Windows   - C:\ProgramData\ssh\sshd_config

Пользовательский файл конфигурации SSH в Windows   - C:\Users\murat\.ssh\config

Приоритет конфигураций SSH   - терминал, пользовательская конфигурация, глобальная конфигурация

chmod 0600 ~/.ssh/config   - установить только права RW владельцу файла config

 

Пример файла конфигурации SSH   -

Host server1

   HostName 172.16.0.11

   User murat

   Port 22

 

Host server2

   HostName 172.16.0.12

   User murat

 

Host server3

   HostName 172.16.0.13

   User murat

   IdentityFile ~/.ssh/serv3-key

 

Host *

   StrictHostKeyChecking no

   UserKnownHostsFile /dev/null

 

IdentityFile ~/.ssh/serv3-key  (SSH config)   - (SSH config) указать приватный ключ ~/.ssh/serv3-key

StrictHostKeyChecking no  (SSH config)   - (SSH config) не реагировать на неопознанный или измененный ключ

UserKnownHostsFile /dev/null  (SSH config)   - (SSH config) не создавать файл known_hosts
